For clarification...

If one selector has more than one rule at the same file, the last file read is always considered and can overrides previous elements.

The "!important" element prevents this from happening by giving the included element precedence

Font-Family

If the font-family has multiple different fonts separated by commas, it means that if the browser does not support the first font on the list, it can use the second alternative font, and if not, the third. If it does not have any of them, the font used will be of the browser default.

Serif: with fancy edges
Sans-serif: no fancy edges (preferred as it is easier to read)

If you downloaded or created your own font-family, use src:url as seen in the code to determine the source of the font file that you want to use.

Don't forget that if the name of the font-family is of multiple words, you have to put it in quotes as seen in the code.
